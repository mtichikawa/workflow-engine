Compose your own workflow

Four commands from clone to a workflow built out of your own words.

1 · Clone & install (no third-party deps)

# the engine is pure standard library
git clone https://github.com/mtichikawa/workflow-engine.git
cd workflow-engine && pip install -e .

2 · Give it a brain (pick one)

# Claude CLI — free on a Max plan, the default. Just install + sign in.
# or the billed API:  pip install anthropic && export ANTHROPIC_API_KEY=sk-ant-…
export ENGINE_BRAIN=api

3 · Compose one of ours

python -m engine.cli compose "read GitHub issues, label and prioritize them, and draft a first response"

It prints the recipe — every step tagged reuse (from the library), NEW! (a gap it flags, not fakes), [GATE] (a human pause) — and whether it's runnable.

Now your turn

Describe any judgment-heavy, multi-step job:

python -m engine.cli compose "<your workflow in plain English>"

4 · Run what it composed

python -m engine.cli compose "screen support tickets and draft replies" \
    --run --input "API returns 500 on every POST since this morning" --auto

— stages a result to state/composed-<run_id>.json. Add --draft to fill gaps with provisional specialists (untrusted — check at the gate).

Run a workflow yourself

Same clone + brain as the Composer steps. Then run any of the three recipes — and point them at your own input.

Run a recipe

python -m engine.cli run triage --repo vercel/next.js --limit 5
inputreal open issues pulled from the repo doesclassify · rank urgency · route · draft reply · verify · stage you geta board summary in your terminal + a run id wherefull log → state/<run_id>.json · staged actions → output/staged.jsonl

The other two

python -m engine.cli run content --topic "why flaky tests erode trust" --brief "sharp technical founder" --auto
python -m engine.cli run refine  --topic "why on-call rotations burn people out" --auto  # draft→grade→rewrite until it passes

Now your turn

Point it at any public repo or any topic:

python -m engine.cli run triage --repo psf/requests --limit 8

Nothing is ever sent — act is gated and only stages. Re-inspect any run: engine board <run_id>.
